# THE BLOCK: GROWTH STRATEGY
## Consumer Social Growth — Intimate, Ethical, Aesthetic

---

> The Block's growth paradox: the thing that makes it valuable (smallness, closeness)
> conflicts with the instinct to grow it fast. Resolve this early or you'll kill the product
> trying to save it.
>
> The answer: don't grow the network — grow the artifact.
> The neighborhood, the rooms, the journey maps ARE the product.
> Make them so beautiful that people bring the outside world to them.

## PART 1: THE GROWTH MODEL

### Why Standard Viral Loops Won't Work

Traditional social apps grow via:
- Public content → discovery → follow → post
- FOMO from seeing other people's metrics
- Broad sharing to prove social worth

The Block has none of these mechanics. That's the point. So we need a different model.

### The Block Growth Model: Artifact Virality

```
Core Loop:
User creates beautiful artifact (room, journey recap, year wrap)
         ↓
Artifact is too beautiful NOT to share externally
         ↓
Artifact appears on TikTok, Twitter, iMessage, home screen
         ↓
Viewer asks: "What is this?"
         ↓
They request an invite from someone who has The Block
         ↓
Invite is scarce → invitation feels special
         ↓
New user onboards → their room appears in inviter's neighborhood
         ↓
Inviter's neighborhood lights up → emotional reward → they invite more

K-factor target: 0.7-0.85
(Not >1 — don't need it. Retention x time = organic growth.)
Retention target: L30 > 55%, L90 > 35%
(These are the real numbers that matter. Most intimate apps die at L7.)
```

### Growth Phases

```
Phase 1 (0-10K users): Seed culture
  → Curate the first 200 users obsessively
  → Every early user gets concierge onboarding
  → Aesthetic is perfect before anyone arrives
  → Goal: Create the "core" culture before it scales

Phase 2 (10K-100K): Unlock artifact virality
  → Journey maps become shareable
  → Widget is ready for home screens
  → TikTok trend seeding begins
  → Goal: Widget seen by 500K people

Phase 3 (100K-1M): Neighborhood density
  → Friend-of-friend discovery (never strangers)
  → City-level neighborhood clustering
  → Goal: Every user has 6+ active friends on day 30

Phase 4 (1M+): Emotional depth moat
  → Memory effects = irreplaceable data
  → Friendship anniversary rituals
  → Goal: User couldn't imagine leaving
```

## PART 2: INVITE SYSTEM

### Design Philosophy
An invite is not a referral link. It's a gift.
If it feels like a referral link, the product is already compromised.

### Invite Architecture

**Invite Scarcity (Tiered by Friendship Level)**

```
New user joins:
  → Gets 3 invite codes immediately
  → Can give to up to 3 people they genuinely care about

Friendship Level 2 (active for 2 weeks):
  → 2 more invite codes unlocked

Friendship Level 3 (active for 1 month):
  → 2 more codes unlocked

Premium member:
  → Unlimited codes (for close friends, not broadcast)
```

Reasoning: Scarcity isn't a dark pattern here — it's a quality filter.
If you'd spam 50 people, The Block is the wrong app for you.
If you'd save a code for 3 specific people, you're the right user.

**The Named Invite**

Instead of a generic link, each invite is addressed:

```
┌─────────────────────────────────────┐
│ you've been invited to              │
│ the block                           │
│                                     │
│ by: alex                            │
│ "saved this spot for you"           │
│                                     │
│ alex's window is waiting to         │
│ light up when you arrive.           │
│                                     │
│ [claim your spot]                   │
│                                     │
│ code: ALEX-7K2M                     │
└─────────────────────────────────────┘
```

This is shareable as a screenshot, a message, a TikTok.
The recipient KNOWS someone was thinking of them. Not automated.

**Invite Delivery Mechanisms**

Priority order (most intimate → least):
1. iMessage / SMS (built-in share sheet)
2. Copy code (paste anywhere)
3. QR code (IRL — most valuable acquisition)
4. Instagram DM story reply

Never: email blast, contact list spam, "invite all contacts" button

**Inviter Reward: The Notification That Matters**

When an invitee creates their room:
```
"alex just moved into the block"
"their window will be lit tonight"
[open the neighborhood]
```

This notification has the highest open rate of any notification you'll ever send.
It's emotional, it's real, it's about a specific person.

**Passive Contact Discovery (Opt-In Only)**

- After 30 days of active use, offer: "want to see if any contacts are on the block?"
- Silent matching on phone numbers (hashed)
- Never show names of people NOT on the platform (reduces FOMO for people not yet on)
- Show: "2 people you know are already on the block. want to connect?"
- User chooses explicitly — never auto-connect

## PART 3: ONBOARDING

### The Onboarding Emotional Arc

The Block's onboarding must answer one question:
**"Is this worth inviting my actual close friends to?"**

By the end of onboarding, the user must feel:
1. This is beautiful and different from anything else
2. My identity is already here (my room = me)
3. I know exactly who I want to invite

```
Onboarding Flow (13 screens, ~4 minutes):

Screen 1: Invite redemption
  "alex saved a spot for you"
  → Enter invite code
  → Personal, not transactional

Screen 2: The neighborhood reveal
  → First time seeing the neighborhood at night (animation)
  → Alex's window glows
  → "this is where your people live"

Screen 3: Phone verification
  → Framed as privacy, not identity verification
  → "just for your people. no one else will find you."

Screen 4: Username
  → "what do your people call you?"
  → Lowercase only, cozy feel
  → No suggestions, no algorithms

Screen 5: Choose your room type
  → Visual room selection (8 types)
  → "where you'll live on the block"
  → Full preview of each room

Screen 6: Room color
  → Pick your wall color, floor, accent
  → Personalization creates ownership immediately

Screen 7: Add your first furniture
  → 3 starter items included
  → Premium items visible but not pushed

Screen 8: Preview your room
  → See exactly how your room looks
  → From outside (how friends see it), from inside
  → "this is home"

Screen 9: Set your mood
  → First mood selection
  → Explain what moods do (change window glow)
  → Not required — "or skip for now"

Screen 10: Notification opt-in
  → Framed honestly: "when do you want to know?"
  → Options: "only when a friend posts" / "gentle daily check-in" / "nothing for now"
  → No dark patterns, no FOMO framing

Screen 11: Widget setup prompt
  → Show what the widget looks like on a home screen
  → "your neighborhood, always nearby"
  → Deep link to widget gallery

Screen 12: Invite 2 friends
  → "who else belongs on the block?"
  → Address by name: "invite [name from contacts]"
  → Easy to skip if not ready

Screen 13: Welcome to the neighborhood
  → First view of live neighborhood
  → Alex's window is glowing
  → "go say hi"
  → Done.
```

### The Activation Moment

Activation = the moment a new user feels "this is MY place."

For The Block, this happens when:
- Their room is complete AND
- A friend's window glows for the first time AND
- They tap the window and see a post

Design this 3-event sequence to happen within the first 7 minutes.
If any of the 3 doesn't happen → user churns.

**Engineering the Activation Sequence:**
- When new user joins, send inviter a push: "they're setting up their room — go say hi"
- Inviter posts (or their existing post surfaces) → new user's first glow
- New user taps, sees the post, feels "this is real and alive"

## PART 4: EMOTIONAL RETENTION SYSTEMS

### The Memory Moat

The longer a user uses The Block, the MORE irreplaceable it becomes.
Every photo, every mood, every journey map creates a memory artifact they can't get elsewhere.
This is the true retention mechanism: **emotional sunk cost** that doesn't feel like a trap.

```
Day 1:   Novelty and curiosity
Week 1:  First journey map auto-generates → "this is beautiful"
Month 1: "On This Day" first fires → emotional hit
Month 3: Friendship anniversary notification → "1 month on the block with alex"
Month 6: First seasonal recap (summer) → exports and shares it
Year 1:  Full year journey → deeply personal artifact they can't delete
```

### Retention Calendar

```
Daily (opt-in only):
  → Daily prompt notification (creative, poetic — not "post today!")
  → "tonight's block moment:" + prompt
  → Sent at user's preferred time (set in onboarding)
  → 5 days/week max (weekends feel different, respect that)

Weekly (automatic, no notification):
  → Weekly journey auto-generated every Sunday night
  → User sees it Monday when they open app
  → "your week in moments" — 1-7 photos, auto-curated

Monthly (notification, can opt out):
  → Monthly recap available on 1st of month
  → Not pushed hard — just a gentle: "your [Month] is ready"

Seasonally (special, celebrated):
  → End of each season: cinematic recap
  → Highest effort, most beautiful presentation
  → Shareable as video — #1 external acquisition moment

Annually:
  → Year in review (January 1)
  → "your year on the block" — full cinematic recap
  → Beautiful enough to share anywhere
  → App Store review prompt fires after user watches it

Friendship anniversaries:
  → "1 year since you and alex became neighbors"
  → Shows first photo shared between you
  → Invite them to reflect together
  → Never: streak guilt ("don't break your streak!")

Memory resurfacing:
  → "1 year ago today..." (only if there's a photo)
  → Fired at 9:00 AM
  → Links directly to that moment in journey map
  → 3x/week max — don't oversaturate
```

### Churn Prevention (The Quiet Neighborhood Problem)

The biggest churn risk: user's friends go inactive → neighborhood feels empty → user leaves.

**Prevention mechanisms:**

1. **"Your neighborhood misses you" — done right**
   - Never guilt-trip. Ever.
   - "a few windows have been quiet lately — good time to check in?"
   - Only fires if user's own friends have been inactive (not the user)
   - Maximum once per 2 weeks

2. **The Ambient Neighborhood Effect**
   - Even without new posts, neighborhood feels alive (weather, time of day, animations)
   - Reduces dependence on friend activity for the app to feel "worth opening"

3. **Solo value proposition**
   - Journey maps work even with 1 friend
   - Daily prompts work alone
   - The user's room and history have value independent of others
   - This is critical: users who churn usually do so because their network died

4. **Friend activity digest (weekly, opt-in)**
   - "your neighborhood this week" — summary of all friend activity
   - Surfaces posts they may have missed
   - Delivered Friday evening (highest open rate)
   - Not a feed — a curated letter

## PART 5: NOTIFICATION STRATEGY

### The Notification Hierarchy

```
Tier 1: Always send (user can only globally disable)
  → Friend joins the neighborhood (invitee you sent code to)
  → Direct reaction to YOUR post

Tier 2: On by default, easy opt-out per type
  → Friend posts (their window glows)
  → Weekly journey ready
  → Memory resurfacing ("1 year ago...")

Tier 3: Off by default, opt-in
  → Daily prompt
  → Friend is online now
  → Monthly/seasonal recaps

Tier 4: Never send
  → "X people viewed your room"
  → "You haven't posted in Y days"
  → Streak warnings
  → Engagement bait of any kind
```

### Notification Language

The language of notifications is the brand.
Every notification should feel like it came from a thoughtful friend, not a growth algorithm.

```
GOOD:
"alex's window is glowing"
"a quiet night on the block"
"this moment, one year ago →"
"your summer is ready to watch"
"maya just moved into the block"

BAD:
"🔥 Don't break your streak!"
"3 friends posted! See what you missed."
"You haven't checked in lately."
"New activity on The Block"
"Tap to see what's happening"

NEVER:
"Your friend Alex liked your photo"  (no likes)
"23 people viewed your profile"      (no metrics)
"You're falling behind!"             (no competition)
```

### Notification Timing Strategy

```
Peak engagement windows (research-backed):
  → 7:30-8:30 AM (morning routine, pre-phone-down)
  → 12:00-12:30 PM (lunch)
  → 9:00-10:30 PM (evening wind-down — highest emotional engagement)

The Block's optimal window:
  → Primary: 9:00-10:00 PM (cozy, winding down, reflective)
  → Secondary: 8:00-8:30 AM (morning ritual)
  → Never: 11 PM - 7 AM (unless user opts in for late-night mode)

User-level personalization:
  → Track when each user opens the app most
  → Deliver notifications 15 min before their typical open time
  → Never deliver more than 2 notifications in a day (hard limit)
  → Weekly notification audit: if open rate < 20%, reduce to 1/day
```

## PART 6: WIDGET VIRALITY

### The Home Screen as Billboard

A widget seen on someone else's phone is the highest-value acquisition touchpoint.
It happens IRL. There's no ad spend. There's no algorithm.
It requires one question: "what's that on your home screen?"

**The widget must be:**
1. Instantly recognizable (distinctive art style)
2. Beautiful enough that people keep it even without the app
3. Different every time you look (living, not static)
4. A conversation starter with anyone who sees it

### Widget Designs

```
Widget 1: The Neighborhood (Large, 4x4)
  → Full neighborhood view, pixel art
  → Night sky, stars, your friends' lit windows
  → Updates when friends post (window glows)
  → Changes with time of day (morning light, golden hour, night)
  → Shows weather state (rain, snow)
  → The most beautiful home screen widget ever made
  → Acquisition potential: HIGHEST

Widget 2: Friend Windows (Medium, 2x4)
  → 4 friend rooms shown in a row
  → Glowing windows indicate new posts
  → Tap any window → deep link to their room
  → Shows friend's current mood as window color

Widget 3: Your Room (Medium, 2x2)
  → Your room from outside, exterior view
  → Your window color = your current mood
  → Tap → set new mood / post
  → Shows: last post thumbnail in window if you posted today

Widget 4: Memory (Small, 2x2)
  → "1 year ago today..."
  → Photo from your archive on that date
  → Changes daily
  → Quietly nostalgic, not clickbait

Widget 5: Daily Prompt (Small, 2x2)
  → Today's prompt text
  → Poetic, minimal
  → Tap → open camera with prompt
```

### Widget Virality Mechanics

**Step 1: Widget is exceptional (earn the share)**
- The neighborhood widget must be genuinely beautiful
- The art quality has to be good enough to be its own aesthetic
- Ship a demo mode: widget previews even without the app (requires careful deeplink handling)

**Step 2: Make it easy to show off**
- "Share your neighborhood" in app → generates a screenshot or short video
- Auto-animates: windows glowing, stars twinkling, time-lapse day-night cycle
- Shareable to Instagram Stories, TikTok, iMessage
- #myblock hashtag built into share copy

**Step 3: Make it findable**
- Widget appears in App Store widget gallery (app preview screenshots)
- "Best home screen widgets" articles will feature it
- r/widgetsmith, r/iphone communities will share it organically if it's good enough

## PART 7: AESTHETIC VIRALITY

### The Core Insight

Instagram grew because it made everyone look like a good photographer.
BeReal died partly because real photos felt boring to share.
The Block must make real photos feel cinematic and meaningful without any filters.

The aesthetic is the product. If the visual identity isn't distinctive enough to be recognizable in 1 second → you have no aesthetic virality.

### The Recognizable Visual Language

**What makes The Block screenshots identifiable at a glance:**

```
1. The neighborhood map at night
   → Pixel art buildings, warm window glow, stars
   → Instantly recognizable visual
   → Nobody else has this

2. The room interior view
   → Cozy pixel-art room, personal touches
   → "Animal Crossing but for memories"

3. The journey map
   → Beautiful typography, cinematic layout
   → Photos in a visual flow with poetic captions
   → Distinct serif font, warm photography treatment

4. The mood indicator
   → Color-coded window glow
   → Orange = cozy, gold = happy, blue = reflective
   → Creates a shared emotional vocabulary

5. The post card
   → Single photo, time/weather/mood shown
   → No clutter, no metrics
   → The photo is the only thing
```

### Shareable Artifacts

These are the things users will share OUTSIDE The Block, driving new user acquisition:

**Tier 1: Season Recaps (highest share rate)**
- End of summer/fall/winter/spring
- Auto-generated video: 60-90 seconds
- Cinematic treatment, lo-fi music, beautiful transitions
- Title in poetic font: "Summer's Quiet Decline, 2025"
- Share rate target: 30% of active users share this

**Tier 2: Journey Maps (medium share rate)**
- "My road trip, 5 days in photos"
- "Our summer, mapped"
- PDF export OR video export
- Beautiful layout → people share as Twitter threads or Instagram carousel
- Share rate target: 15% of journey map viewers share

**Tier 3: Year in Review (highest emotional share)**
- January 1st every year
- "2025 on the block"
- This is the Spotify Wrapped equivalent
- Target: every user opens this, 25% share externally

**Tier 4: Room Screenshots**
- Users screenshot their room + share as Discord avatar, Twitter header, phone wallpaper
- This requires the room art to be exceptional quality
- "Made in The Block" watermark (tasteful, can be removed) → brand recognition

### Aesthetic Consistency Rules

For aesthetic virality to work, the visual identity cannot drift.

```
Rules:
✓ Every screenshot must be recognizable as The Block
✓ Color palette: always warm-dark (never bright white backgrounds)
✓ Typography: consistent serif for titles, Inter for body
✓ Photo treatment: no filters, but beautiful composition framing
✓ Pixel art quality: must be as good as indie game art
✓ Empty states are beautiful, not blank

Never:
✗ Inconsistent UI (feature teams shipping mismatched screens)
✗ Bright backgrounds on shareable content
✗ Corporate-looking export frames
✗ Cheap watermarks
```

## PART 8: TIKTOK VIRALITY

### The Platform Reality

TikTok is the only platform where an unknown app can reach millions for free.
But TikTok content about apps is dominated by:
- "This app is dangerous" (fear)
- "This app will get you free followers" (greed)
- "This app is WILD" (shock)

The Block doesn't fit any of those. Which is exactly the opportunity.

### The Block's TikTok Angle: Quiet Intimacy in a Loud World

The contrast between TikTok's chaos and The Block's coziness IS the content.

### Content Categories That Will Work

**Category 1: "Show me your room"**
```
Format: Screen recording of your Block room
  → Custom room tour, showing each furniture piece
  → What each item means to you
  → "this corner is for when I'm feeling reflective"
→ Comments: "how did you make it look like this??"
→ Acquisition trigger: "what app is this?"

Seed this with 10 creator accounts in Month 1
One good room tour = 500K-2M views easily
```

**Category 2: "My neighborhood at 3am"**
```
Format: Phone screen showing neighborhood widget at night
  → Stars, quiet windows, maybe 1 lit window
  → Voiceover: "my friend is awake at 3am..."
  → Tap their window, see their photo
  → Emotional, real, intimate
→ This hits the "this is so real" emotional TikTok trigger
→ The late-night intimacy angle is genuinely compelling content
```

**Category 3: Season recap reveals**
```
Format: "Reacting to my year on The Block"
  → Watch the auto-generated recap video
  → Real emotional reactions
  → "I forgot about this day..."
→ Authentic nostalgia content → extremely high engagement
→ Mirror format (trending): before vs after emotional state
```

**Category 4: "This social media has no likes, no followers"**
```
Format: Explain The Block's anti-algorithm design
  → "there are no likes on this app"
  → "no follower counts"
  → "you can only post 1 photo a day"
→ Gen Z WANTS this content — they're burned out on metrics
→ "is this real??" comment triggers → massive organic spread
→ This video has been made about BeReal — but The Block's version is more defensible
```

**Category 5: "I invited my 3 best friends, here's what happened"**
```
Format: Document the first 30 days of you + 3 friends on The Block
  → Day 1: setting up rooms
  → Day 7: first journey map
  → Day 30: "it changed how we stay close"
→ Serialized content → subscriber growth
→ Authentic relationship content (Gen Z's most trusted content type)
```

**Category 6: The Contrast Video**
```
Format: Split screen
  → Left: Instagram (likes, follower counts, pressure)
  → Right: The Block (just photos, just friends, just vibes)
→ "I quit Instagram for 30 days and used this instead"
→ Mental health angle is huge on TikTok right now
→ Non-clickbait because it's actually true
```

### TikTok Seeding Strategy

**Month 1-2 (before public launch):**
- Identify 15-20 micro-creators (50K-500K followers) in:
  - Aesthetic/cozy content
  - Mental health / digital wellness
  - "App of the week" / tech discovery
  - Gen Z friendship content
- Give them early access + all invite codes they want
- Brief: "use it naturally with your real friends for 30 days"
- NOT: "promote The Block" — that kills authenticity
- Let them find their own angle

**Month 3+ (growth phase):**
- UGC amplification: reshare the best user content
- Stitch responses to "what app is this?" comments
- Creator program: top 20 creators get early feature access
- Never pay for sponsored posts (kills the authentic brand)

### What NOT to Do on TikTok

```
✗ Don't make an official account that posts polished brand content
✗ Don't run ads on TikTok (feels wrong for the brand)
✗ Don't chase trends that don't fit (no forced audio, no dancing)
✗ Don't respond defensively to criticism
✗ Don't promise features you don't have yet
```

## PART 10: GEN Z RETENTION

### Why Gen Z Is the Right Audience

Gen Z has:
- The highest digital burnout of any generation (76% report social media anxiety)
- The highest appetite for "real" and "authentic" content
- The strongest aesthetic identity investment (Animal Crossing, Notion setups, Pinterest boards)
- The most developed sense of anti-corporate values
- The most to gain from an intimate social app (their friendships are under-served by existing apps)

They're not the demographic to acquire. They're the demographic to serve.

### Gen Z Values Alignment

```
Gen Z value              How The Block addresses it
─────────────────────────────────────────────────────
Anti-algorithm           No feed. Intentional visits.
Mental health awareness  Session length nudges. No addiction mechanics.
Aesthetic identity       Room customization. Cozy visual system.
Authenticity             Real-time camera only. No filters.
Anti-corporate           Invite-only. No ads. Premium cosmetics only.
Nostalgia                Journey maps. Memory resurfacing. Year recaps.
Intimacy over scale      12-15 friends, not 500 followers.
Digital home             Room metaphor. Animal Crossing energy.
Control                  Full privacy controls. Delete anytime.
Meaningful metrics       Friendship levels (depth), not follower counts
```

### Retention Mechanics That Work for Gen Z

**1. Aesthetic Investment = Retention**

The room is the retention mechanism. The more customized it is, the harder it is to leave.

- Week 1: Basic room set up
- Month 1: Premium furniture discovered
- Month 3: Seasonal decoration added
- Month 6: Full room identity established
- Year 1: The room IS them — deleting the app means losing this

Ship room customization updates every 6 weeks (new furniture sets, seasonal items).
Each update is a reason to re-engage.

**2. Friendship Level Progression (Non-Competitive)**

```
Friendship levels between two users:
  Level 1: Neighbors (default on connection)
  Level 2: Friends (2 weeks of mutual activity)
  Level 3: Close Friends (1 month of regular interaction)
  Level 4: Besties (3 months of consistent closeness)
  Level 5: Ride or Die (6+ months of deep connection)

Each level unlocks:
  → New room decorations for that friendship
  → Shared album capacity increases
  → Special lighting effects when they post
  → A small ceremony notification ("you and alex just became close friends")

Critical design rule:
  → Never show leaderboards
  → Never compare friendship levels across friends
  → Never create competition between friends
  → Levels are descriptive, not prescriptive
```

**3. Seasonal Events (FOMO Without Addiction)**

```
Spring event: "Block Bloom"
  → Cherry blossom decorations available
  → Spring prompt pack (15 new prompts)
  → Neighborhood gets spring visual treatment
  → Duration: 3 weeks
  → After 3 weeks: decorations stay, new ones don't appear until next year

Summer event: "Golden Hour Week"
  → Every evening, neighborhood renders in golden hour lighting
  → Special golden post frame (optional, tasteful)
  → Summer recap auto-generates at end

Fall event: "The Quiet Season"
  → Amber and falling leaf decorations
  → Fall mood pack
  → "reflective season" prompt themes

Winter event: "Block After Dark"
  → Snow on rooftops
  → Christmas lights on some buildings (opt in/out per user)
  → "cozy season" — most important retention period
  → New Year memory ceremony

These events work because:
→ They're time-limited (creates organic urgency)
→ They're beautiful (people open app just to see the seasonal look)
→ They're not monetized (access is free — premium cosmetics may be themed but events are open)
→ They create shared memories tied to seasons
```

**4. The "Good Enough" Retention Rule**

Gen Z will leave if they feel manipulated.
The Block's retention should feel like visiting a neighborhood you love, not checking a Snapchat streak.

Test: Would a user tell a friend about The Block's engagement system in a positive or negative light?

```
Positive framing (keep):
"It showed me a photo from a year ago and I cried"
"My room changes with the seasons"
"My friend's window was glowing at midnight and I just knew they were thinking"

Negative framing (remove immediately):
"It told me to post or I'd lose my streak"
"It showed me how many days since I last opened it"
"My engagement metrics dropped when I didn't use it"
```

## PART 11: GROWTH METRICS

### What to Measure (and What NOT to)

```
Measure:
✓ L7, L30, L90 retention (most important)
✓ Time between friend invite and invitee onboard
✓ Widget install rate
✓ Journey map open rate after auto-generation
✓ Notification open rate by type
✓ Season recap share rate
✓ "On This Day" tap-through rate
✓ Room customization depth (furniture items placed)
✓ Avg friend count per user on Day 30

Don't measure (or at least don't optimize for):
✗ Daily Active Users (can be high and app still be extractive)
✗ Session length (doesn't reflect quality)
✗ Post volume per user
✗ Notification CTR at any cost (easy to inflate, destroys trust)
✗ Viral coefficient in isolation (focus on K-factor × retention)
```

### North Star Metric

**"Meaningful Neighborhood Density" (MND)**

Definition: % of users who have 4+ active friends in their neighborhood on Day 30.

Research shows: Users with 4+ active friends have L90 retention of 55%+.
Users with 0-1 active friends have L90 retention of 8%.

Everything in growth should optimize for this one number.

```
How to improve MND:
→ Reduce time-to-first-friend-join after invite
→ Surface quiet friends ("alex hasn't posted in 2 weeks — maybe check in?")
→ Make friend inviting easy and emotionally motivated
→ Remove users who are dead weight (inactive 90+ days, with notice)
```

### 12-Month Growth Targets

```
Month 1:  1,000 users   (curated seed cohort — quality over quantity)
Month 3:  10,000 users  (artifact virality beginning, widget on home screens)
Month 6:  50,000 users  (TikTok seeding paying off)
Month 9:  200,000 users (first season recap goes viral)
Month 12: 500,000 users (year-in-review moment — organic press)

L30 target: >55% throughout (if this drops, pause growth spend, fix product)
MND target: >60% throughout
```
